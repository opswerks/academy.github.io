Lab exam part1

Linux Administration Practical Assessment Part 1

Before you begin...

  • Make sure you can login to your instance using your account and password (see here for reference)
  • Note: Your instances will be rebooted after you have taken your exam

Tasks

Task 1: Essential Tools

  1. (10 pts) Create a new user student with the home directory /home/student.
  2. (10 pts) Create a copy of /etc/dnf/dnf.conf and save it in /home/student/documents/projects/project1.txt. Then, create a copy of /etc/ssh/sshd_config and name it /home/student/documents/projects/project2.txt
  3. (10 pts) Create a hard link to the file /home/student/documents/projects/project1.txt in the /home/student/links directory, named project1-hardlink.txt. Then, create a soft link to the same file in the /home/student/links directory named project1-softlink.txt.

  4. (20 pts) Set the following permissions:

    • /home/student/documents/projects/project1.txt should be readable and writable by the owner, readable by the group, and not accessible by others
    • Set the SUID bit on the file /home/student/documents/projects/project2.txt
    • Set the SGID bit on the directory /home/student/documents
    • Set the sticky bit on the directory /home/student/documents/projects

  5. (10 pts) Search for all files with a filename ending with .conf in the /etc directory and save the list of these files to/home/student/conf_files.txt. Then, find all occurrences of the word "network" in these files and save the results to /home/student/network_mentions.txt.

Task 2: BASH Scripting

  1. (15 pts) Write a shell script named /home/student/backup_home.sh that automates the process of creating a backup of the /var directory. The script should:
    • Create a tarball of the /var directory with a filename that includes the current date. The following filename format must be followed: /home/student/var_backup_YYYYMMDD.tar.gz (e.g. var_backup_20240128.tar.gz)
    • If the backup was successful, print a message Backup successful.
    • Heres how it should look like when executed, and if you list the files under students home dir: 
      $ /home/student/backup_home.sh 
              Backup successful 
              $ ls /home/student/var_backup_* 
              var_backup_20240729

  2. (15 pts) Create a shell script named /home/student/user_check.sh that checks if a given username (passed as an argument) exists on the system. The script should:

    • Print a message indicating whether the user exists or not: User exists / User does not exist
    • The scripts exit status must be 0 if the user exists, and the exit status must be 127 if the user does not exist
    • Heres how it should look like when executed: 
      $ /home/student/user_check.sh root 
User exists 
$ echo $? 
0 
$ /home/student/user_check.sh nonexistentuser 
User does not exist 
$ echo $? 
127

  3. (15 pts) Write a shell script named /home/student/disk_usage.sh that processes disk usage information of the /boot directory. The script should:

    • Print the disk usage of each subdirectory within /boot in human-readable format
    • Sort the output ascendingly by directory size, starting from the directory with the smallest disk usage up to the directory with the largest disk usage
    • Lastly, the script must save the output of the disk usage information to a file named /home/student/disk_usage_report.txt

Task 3: Operate Running Systems

  1. (10 pts) Spawn a sleep 100000 process that should be running in the background. The process must still be running in the background even if you log out your session. Then, make sure its niceness is set to 10.

Task 4: Configure Local Storage and Filesystems

  1. (5 pts) Create a partition in /dev/sdb with a size of 128MB
  2. (10 pts) Format /dev/sdb1 to ext4 and mount it to /mnt/data. Ensure that the mount is persistent and will be mounted during system bootup.
  3. (5 pts) Create two more partitions in /dev/sdb with a size of 64MB each
  4. (10 pts) Create a software RAID group (RAID1) out of /dev/sdb2 and /dev/sdb3. Format the resulting RAID group in xfs and mount it to /mnt/raid1. Ensure that the mount is persistent and will be mounted during system bootup.
  5. (20 pts) Create one more partition in /dev/sdb with a size of 128MB. Ensure that partition /dev/sdb4 is using the Linux LVM partition type. Create a physical volume out of it, then create volume group called vgsdb out of the physical volume. Then, create a 100MB logical volume named lvsdb from vgsdb. Lastly, Format the logical volume to ext4 and mount it to /mnt/lvm. Ensure that the mount is persistent and will be mounted during system bootup.

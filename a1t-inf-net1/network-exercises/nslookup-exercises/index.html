Practice Exercise: DNS Troubleshooting with Linux nslookup

Introduction

In this lab, you will play the role of a system administrator working with DNS (Domain Name System) on a network consisting of two CentOS 8 machines: one acting as a DNS server and the other as a client. DNS is a critical component of network infrastructure, responsible for translating human-readable domain names into IP addresses. This lab will teach you how to use the nslookup command for DNS troubleshooting and configuration.

Prerequisites

  • Basic knowledge of Linux command line
  • Two CentOS 8 virtual machines (one server, one client)
  • Network connectivity between the two machines

Lab Scenario

You are responsible for managing the DNS server for your organization's network. The client machine relies on this server for DNS resolution. Your goal is to troubleshoot DNS-related issues, configure DNS records, and ensure smooth communication between the client and server.

Tasks

Task 1: DNS Server Setup

  • Install and configure a DNS server (e.g., BIND) on the server machine.
  • Create a basic zone file for a sample domain (e.g., example.com).
  • Start the DNS server service.

Task 2: DNS Client Configuration

  • On the client machine, configure DNS resolution to use the DNS server.
  • Verify that the client can resolve domain names using the DNS server.

Task 3: Basic nslookup Usage

  • On the client machine, use nslookup to resolve the IP address of a well-known domain (e.g., google.com).
  • Analyze the output and understand the components.

Task 4: Resolving Local Domain

  • Use nslookup on the client to resolve a domain within your organization (e.g., subdomain.example.com).
  • Verify that the DNS server correctly resolves this local domain.

Task 5: Reverse DNS Lookup

  • Perform a reverse DNS lookup on an IP address using nslookup.
  • Confirm that you can find the domain associated with the IP.

Task 6: MX Record Lookup

  • Use nslookup to retrieve the mail server (MX) record for a domain (e.g., example.com).
  • Understand the priority and mail server information.

Task 7: Troubleshooting DNS Server

  • Simulate a DNS server failure on the server machine (e.g., stop the DNS service).
  • On the client machine, attempt to resolve a domain and troubleshoot the error message received.

Task 8: DNS Record Modification

  • Modify a DNS record on the server (e.g., change an A record).
  • Use nslookup on the client to verify that the DNS server reflects the changes.

Task 9: Caching

  • Perform an nslookup query on the client for a domain.
  • Check if the DNS server caches the response.
  • Modify the DNS record's TTL (Time to Live) on the server and observe the cache behavior.

Task 10: Troubleshooting DNS Client

  • On the client machine, simulate a misconfiguration by setting an incorrect DNS server.
  • Use nslookup to diagnose and correct the client's DNS configuration.

Troubleshooting Command: Simulate a DNS server failure

  • Stop the DNS service on the server machine.
